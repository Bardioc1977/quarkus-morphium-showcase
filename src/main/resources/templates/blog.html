Blog System

Versioning, references, lifecycle callbacks, timestamps and container types

Morphium Features Demonstrated

@Version @CreationTime @LastChange @LastAccess @Reference @Reference(lazy) @Embedded (Comments) @Lifecycle @PreStore @PostLoad List/Map containers @Transient

Entity Definition

@Entity(collectionName = "blog_posts")
@Lifecycle
public class BlogPost {
    @Id private MorphiumId id;
    @Version private Long version;          // optimistic locking
    @CreationTime private LocalDateTime createdAt;
    @LastChange private LocalDateTime lastChanged;
    @LastAccess private LocalDateTime lastAccessed;
    @Reference private Author author;       // eager reference
    @Reference(lazyLoading = true) private Author reviewer;
    private List<Comment> comments;       // @Embedded list
    private List<String> tags;            // simple list
    private Map<String, String> metadata; // map container
    @Transient private String editNote;    // not persisted
}

Service Code (Morphium API)

// Store with @Version (optimistic locking)
morphium.store(post);
// On concurrent modification:
try {
    morphium.store(post);
} catch (VersionMismatchException e) {
    // version conflict - reload and retry
}

// @Reference resolution (eager by default)
Author author = morphium.createQueryFor(Author.class)
    .f(Author.Fields.id).eq(new MorphiumId(authorId))
    .get();
post.setAuthor(author); // stored as DBRef

// Embedded Comment list (stored inline)
post.getComments().add(comment);
morphium.store(post);
