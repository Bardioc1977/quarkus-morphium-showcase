This demo uses a custom ChatMessage entity persisted via Morphium. Messages are stored in MongoDB, filtered by topic using createQueryFor(), and updated in-place with query.set() for targeted field changes (e.g. marking as read). This is not Morphium's built-in Messaging system — it demonstrates the general-purpose query and update API.
