MicroProfile Health

Automatic liveness, readiness and startup probes for Kubernetes via Morphium driver stats

Quarkus Extension Feature

@Liveness @Readiness @Startup MicroProfile Health SmallRye Health

Adding quarkus-smallrye-health to your project automatically activates Morphium's health checks. No code required -- the quarkus-morphium extension registers all three probes via HealthBuildItem at build time. Disable with quarkus.morphium.health.enabled=false.

Liveness Check

Purpose: Is the Morphium driver still connected?
DOWN = Pod should be restarted by Kubernetes.

{livenessStatus} /q/health/live 
@Liveness
morphium.getDriver().isConnected()

Readiness Check

Purpose: Is the driver connected and able to serve requests?
DOWN = No traffic routed to this pod.
Pool stats are reported as metadata but do not affect UP/DOWN.

{readinessStatus} /q/health/ready 
@Readiness
morphium.getDriver().isConnected()
// Pool stats (informational, not affecting status):
// connectionsInUse, connectionsInPool, threadsWaiting, errors

Startup Check

Purpose: Was the initial connection established?
DOWN = K8s defers liveness/readiness probes.

{startupStatus} /q/health/started 
@Startup
isConnected()
  + connectionsOpened count

Configuration

# pom.xml -- just add SmallRye Health, Morphium checks register automatically
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-smallrye-health</artifactId>
</dependency>

# application.properties -- disable all Morphium health checks (optional)
quarkus.morphium.health.enabled=false

MicroProfile Health JSON Endpoints

These standard endpoints return JSON consumed by Kubernetes probes and monitoring tools.

EndpointPurposeK8s Probe
/q/health All health checks combined --
/q/health/live Liveness probes only livenessProbe
/q/health/ready Readiness probes only readinessProbe
/q/health/started Startup probes only startupProbe

Morphium Driver APIs (used by health checks)

// Liveness: Is the driver connected?
boolean connected = morphium.getDriver().isConnected();

// Readiness: Pool statistics
Map<DriverStatsKey, Double> stats = morphium.getDriver().getDriverStats();
double inUse   = stats.get(DriverStatsKey.CONNECTIONS_IN_USE);
double inPool  = stats.get(DriverStatsKey.CONNECTIONS_IN_POOL);
double waiting = stats.get(DriverStatsKey.THREADS_WAITING_FOR_CONNECTION);

// Per-host connection distribution
Map<String, Integer> hosts = morphium.getDriver().getNumConnectionsByHost();
